# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | ✅ Yes    |

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public issue**.

Instead, report it through one of the following methods:

1. Use GitHub [Security Advisories](https://github.com/seino/irodori/security/advisories) to report privately
2. Include the following information in your report:
   - Description of the vulnerability
   - Steps to reproduce
   - Impact assessment
   - Suggested fix (if available)

## Response Process

1. **Acknowledgement**: We will confirm receipt within 48 hours
2. **Investigation**: We will assess the impact and determine a fix strategy
3. **Fix**: A patch will be developed and a fixed version released
4. **Disclosure**: Vulnerability details will be published after the fix is released

Thank you for helping keep irodori safe.
